Edit Application Interface

The Edit Application Interface allows you to change the Application Name and Program File as well as enabling or disabling the Application and setup the code to run when the Application becomes active.

How To Use

Each application you add to the system enables Hot Key Speak for that application.

The application branch shows the application icon extracted from the application's executable and the application name shown was determined by the application's main Window Title that was active at the time you added it using the Process Selector tool.

You can change the application name by editing the branch, you can also change the program itself which is useful to re-grab an application that changed folders, or to set a program for a branch you imported, or if you are making a rule set for a game, you can temporarily swap the application to another program for testing. You can set it to target Notepad for example which is a useful test platform to testing your code.

NOTE

Do not modify the content of the Windows Apps Application Entry in any way! ...Well aside the Enabled Check. This special application item is created when you add a Windows 10 App to the application list. The system relies on this special item to manage Windows 10 Apps.

Name

Specify the name for the Application as it would appear in the tree-view.

Program File

Specify the program file, path and all for the Application.

Use the button on the right to select a new Application with the Select Application Interface.

Enabled

Specify if the Application is enabled. Disabled Applications will not be activated when their associated program is active on the desktop.

Code

Specifies the code to run when the application becomes active. This is a good place to setup global variables and define global functions that the other actions may use.

Use the button to execute the code. This is helpful but not always usable depending on the code you are writing. The console window will pop up to handle any send commands.

Use the button to edit the code in an external editor. An external editor such as MS Visual Code or Notepad++ is recommended. The system should automatically update any changes made by the external editor when you save the file. MS Visual Code has an Auto-save Feature that makes this process seamless.

Use the button to set the external editor to the program you want to use.

Use the button to reload the code from the temporary file the external editor is editing if the automatic update is not working.

OK Button

Applies the changes and closes the dialog box. Some changes may require a restart.

Cancel Button

Cancels the changes and closes the dialog box.

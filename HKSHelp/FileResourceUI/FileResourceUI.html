Explore File Resource Interface

This standard Save File dialog, launched when you edit a File Resource, is used to export the selected file resource to a file so you can investigate it yourself. There is no actual Edit File Resource Interface per se since changing a resource is as simple as deleting the old one and adding a new one via the right click menu.

How To Use

Type the filename to extract to, make sure the extension matches the original resource.

Upon saving the Windows File Explorer will launch showing the directory you saved the file to.

You can then validate the file performing a virus scan and using the appropriate application for that specific resource to ensure it is what is purports to be.

NOTE

A Resource File is a file attached to the distribution of a HKS script/action definition for a program. Its use is to accommodate adding little extras like small wav files or other things that may help improve user interaction.

The method is paired with the HKS.Resource.Fetch() code feature along with the very few allowed file IO functions, such as SoundPlay() to make use of such files.

Though internally the code system attempts to limit the file IO functions to somewhat naively prevent hacking, the File Resource System could be used to install malware.

So this Edit/Explore File feature is there for you to safely extract a resource file so you can validate that it is what it is. It is up to you to validate and trust the resource files package with projects.

File Name

Specify the file name to extract the file resource to, respecting the file extension. Obviously don't select or type a file that exists unless you don't mind overwriting it.

Save Button

Save the file and launch File Explorer to browse the file.

Cancel Button

Cancels the dialog box.

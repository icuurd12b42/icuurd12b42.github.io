Edit Window Context Interface

The Edit Window Context Interface allows you to change the Window Context Name, the filters (window title, class, active control name and text) to use to detect the window, as well as enabling or disabling the Window Context.

How To Use

A Window Context works like a Command Context, however, it is activated whenever a specific window is active. Such as for example when a File Open Dialog Box is detected

This limits the available options to a stricter set of possibilities, therefore reducing reaction time for the system.

It also allows for a more natural work flow as the user can issue a OK or a Cancel command, when a File Dialog is active for example.

With the various fields available here you can make the system detect anything from a main window, a pop up dialog or even if a specific field or button is active.

NOTE

Unlike a Command Context, only the actions located underneath a Window Context are active.

A Window Context is also considered the root parent of any command or command context listed underneath.

The only way out of a window context is for another window to become active either by user interaction, closing the window or selecting another, or by having your code closing the window or switching to another.

Find Window

This allows you to find a Window using the Desktop.

After Activation, Navigate to the Application Window on the Desktop you wish to add and tap the Left Ctrl on the Keyboard. Some controls may require you hold down the left mouse button on them before you tap the Left Ctrl.

This will populate the fields below.

In the case you used Find Window to fill the fields, be sure to trim the fat from the Window Title, Control Name and Control Text fields; for example, changing c:\somefolder\somefile.txt - Notepad to simply Notepad may be a good idea to find all instances of Notepad

Name

Specify the name for the Window Context as it would appear in the tree-view.

Window Title

Specify the window title to use as filter to be more accurate in the window detecting.

If the field is blank or is set to * it means the field is ignored.

The field may contain a specific piece of text, say Notepad, and the system will find the window who's title contains Notepad anywhere in its title which may be convenient in most cases.

The field may contain a regex based wild-card, not to be confuse with old * and ? based wild-card used to search folders. This allows stricter searches.

Window Class

Specify the Window Class to use as filter to be more accurate in the window detecting.

See Window Title Above for more information.

Control Name

Specify the Active Control Name in the Active Window to use as filter to be more accurate in the window detecting.

See Window Title Above for more information.

Control Text

Specify the Active Control Text in the Active Window to use as filter to be more accurate in the window detecting.

Depending on the Control Type, this may be text in an edit box or a caption in a button...

See Window Title Above for more information.

Enabled

Specify if the Window Context is enabled. Disabled Window Context will be ignored by the system and so will any Commands located under it in the tree structure.

Code

Specifies the code to run when the Window Context is activated. You can call any function or use any variables defined by the code in the Application Interface, or any of the code that was previously ran by items up the context chain. This is also a good place to reset variables that the actions under this context would need.

Use the button to execute the code. This is helpful but not always usable depending on the code you are writing. The console window will pop up to handle any send commands. For Testing purpose, you have to manually run the code of all related actions up the chain first If your code box calls a function or uses variables defined in the Application Interface for example, make sure to run that code first.

Use the button to edit the code in an external editor. An external editor such as MS Visual Code or Notepad++ is recommended. The system should automatically update any changes made by the external editor when you save the file. MS Visual Code has an Auto-save Feature that makes this process seamless.

Use the button to set the external editor to the program you want to use.

Use the button to reload the code from the temporary file the external editor is editing if the automatic update is not working.

OK Button

Applies the changes and closes the dialog box. Some changes may require a restart.

Cancel Button

Cancels the changes and closes the dialog box.

Right Click Context Menu

The Right Click Context Menu is the main work horse of the application. Through it you will add and edit tree-view items that will define the Hot Keys and Speech Recognition behavior for all your applications and games.

How To Use

Hitting Ctrl+Enter or Right Clicking on a tree-view item will pop up a menu that will display a list of actions to perform.

Some menu items will be enabled while other won't. It all depends on the context of the tree-view item clicked (hence the name).

Hitting enter in the menu will trigger the selected item. If no selected item is set, the bold menu item will run by default.

Double clicking on the tree-view item will trigger the Edit menu item.

Below is the list of all the menu actions and what they do.

Add Application

This will launch the Select Application Interface which will allow you to add an application to your list of monitored application under which you will later define the hot key and speech recognition structure.

Applications can be added under the Applications and Games tree-view item or any Group that are off of that root item.

Once an application is added, you can edit its settings via the Edit menu which will launch the Select Application Interface.

Windows 10 Apps will share a single Windows Apps root Application item under which any new Windows 10 App you add will be created in the form of a Window Context item.

Add Command

This will create a New Command Action item and launch the Command Action Interface which will allow you define a hot key and speech recognition phrase along with the code to execute when the action is performed.

Add Context

This will create a New Context Action item and launch the Context Action Interface which will allow you to segregate speech and hot key detection by what can be defined as exploratory context where a series of actions results in digging in the command structure in so that you can contextualize what actions are possible depending on passed user actions.

Add Window Context

This will create a New Window Context Action item and launch the Window Context Action Interface which will allow you to segregate speech and hot key detection by specific active windows or active controls in the application and so that you can contextualize what actions are possible under which dialog box the application presents the user, for example.

Add Group

This will create a New Group item and launch the Group Interface which will allow you to structure the tree-view in a way that makes sense to you. Since Command actions are sorted, searched and prioritized alphabetically, Groups also help defining what actions can be processed first, specifically in the case were a Window Context that targets a specific window with specific data needs to be processed before a similar Window Context that catches a more generic version of that same window.

Add Resource

This will create a New File Resource item by launching a file explorer where you will select a (small) file like a wav file to include in the hot key rules.... Such files are useful to add extra features like sound when an action performs... The system should not be abused by adding files that are too large...

Edit

This will edit the item selected allowing you to make changes to it.

Cut (Prep for Move)

This will flag the item or group/structure (and all items underneath) for a possible subsequent Paste-Move action.

Copy (Prep for Clone)

This will flag the item or group/structure (and all items underneath) for a possible subsequent Paste-Copy action.

Paste (Copy/Move To)

This will perform the copy or move function for the item or structure that was previously flagged by the Cut or Copy functions... to the selected location.

The cut copy paste feature is a little unconventional but you get used to it.

Note that this item will be enabled only if the content that was flagged for copy/cut is compatible with the item or structure you are pasting to. You cannot paste an Application Item under a Window Context for example.

IMPORTANT: You may need to use the Scan For Issues menu after a Paste action in order to fix any parenting issues that may have resulted from the action.

Save to File

This will save the selected item or structure to file so you can share your setup with other people.

Depending on the content you are saving there are 3 possible file types that can be generated....

simphks
Simple Hot Key Script file - contains items or groups which can be imported under an application. This is useful for saving features you re-use often.
exehks
Executable Hot key Script - contains one application or a group of applications which can be imported back under the Applications and Games root item (or a group underneath it). This is basically the way to share your scripts for specific games.
appshks
Applications (and games) Hot Key Script - contains a full Hot Key Script definition file. Importing this would replace your current file completely.

Load From File

Loads a file that was previous saved (see Save above).

You will be warned if you are trying to import a file type that is incompatible with the target item. See Save above for where things are imported.

IMPORTANT: You may need to use the Scan For Issues menu after importing a file in order to fix any parenting issues that may have resulted from the action.

IMPORTANT: You may need to re-assign applications via the Application Interface for application rules you import as they may not be at the same location recorded in the file.

Scan For Issues

This will scan the structures for incompatibilities such as duplicate items and function names and repair broken parenting hierarchy.

Delete

This will delete the selected item and all its children items if applicable.

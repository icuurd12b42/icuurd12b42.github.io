Introduction

This program is the hard work of one aging gamer and developer who has lost a lot of his finger flexibility, getting old sucks.

When I started with gaming in the early 90's I was a pretty good, my reflexes were above average and I ruled most games I played.

Going back into gaming, I found myself lacking the finger flexibility and also the memory to hit the right key at the right time... without hitting something else at the same time.

Today is a a glorious day for gaming but for people like me a lot of it is ruined by our disabilities, so I decided to make a program to compensate so I could at least be a little more competitive as opposed to getting slaughtered by them darn kids!

Hot Key Speak

Hot Key Speak was initially written in order to help people with a loss of finger flexibility and allow them to use their voice to compensate. The idea is that I could control my player with wasd for example and ask the system to reload my gun, or to switch weapons or to throw a grenade.

I needed a system that was local to the PC and fast enough to react in a respectable amount of time.

MS SAPI provided the answer to voice recognition and AutoHotKey provided the means to control the game.

I started with a proof of concept, using some code I found on the AutoHotKey forum that allowed setting up MS SAPI to detect keywords and map them to keystrokes.

I wrote a small program that would define words and keys to send for my favorite multi-player game and it worked beautifully but in a limited fashion.

Then I got another game and created another version of the code for that game... things started to fall apart because that other game required more flexibility and context based action. It became clear that my system needed to do more things and react more intelligently. The commands had to be defined in such a way that I could have a varying degree of context.

One problem I hit was the number of commands I had to add... as many actions would essentially be the same wording (in concept) in game, for the system I had I could not have the same word do multiple things. The result was a ridiculous number of global commands, most of which I could not remember in game.

Another problem was that my generic send method was not enough in many cases, I needed to do more, the actions needed to do more, they needed logic to execute, not just sending keystrokes blindly.

Another problem was that I needed to batch actions as on numerous occasions I would need to tell the system to do a number of things in one sentence, like reload, switch guns, power shield, reload, switch guns. And doing this with long sentences was impossible, and having to wait for the recognition system to figure things out from the huge list before I issued the next command was problematic..

Finally the last problem was that I needed a single program to handle multiple games. I simply could not have a half dozen AutoHotKey Scripts, one for every game, each with their own speech rules and code. Too much of a hassle to start the right one, close it and start another AHK script every time I switched games. Plus if I made the error of leaving another script running it would interfere with the other, specifically on the MS SAPI level.

Features

After almost a year of writing code and testing concept I finally got something that has all the features that will allow anyone to write Voice Activated Scripts for almost any game

Application Detection

The program detects the application/game that is active for you. You don't have to worry about application context.

Window Context

The program detects the active window for you. You can define what Window a context applies to.

Voice Recognition

The program allows you to define what words or sentences trigger an event.

Hot Key Definition

The program allows you to define what hot keys trigger an event.

Command Context

The program allows you to define command context which allows defining actions that are only applicable in certain situations and so you can use the same words or hot keys to do multiple things.

Improved MS SAPI features

The program allows you to define voice rules and can perform actions while you are speaking. Commands can run as soon as the system is sure enough of what you said without you needing to stop talking. The command sentences you define in the rule allow for multiple utterances. Commands will be detected even if they are mixed in with a frustration rant... Believe me I made sure of it :)

Batched Commands

The program allows you issue many orders in one sentence, they will be performed one after the other as the system recognizes the orders given

Discord Compatible

Matching Hold To Speak system so you can talk with friends without the system being active.

Coded Actions (JavaScript)

Well actually JScript, the system allows you to code actions using a JavaScript form of the AutoHotKey API as well as other APIs included in the package.

Every action has code associated with it which allows you to define the behavior in context of the commands issued.

Rule Sharing

You can share game rules and tool rules with friends.

Testing

The system allows you to test the hot keys and the voice recognition without the target game running.

Debugging

The system allows you run the code you wrote for actions without the target game running. The system also has a console window which allows you to output information either from your code or by directly typing in the console.

HotKeySpeak's intent with regards to Gaming is NOT to give you an easy way to create cheating code for game, the initial intent of this program was to have a system that would easily allow people to compensate for their disabilities, thought ultimately this system can also be used to improve your gaming performance. Initially, with the system only responding to voice, there would be no way this system would give anyone the upper hand, but since I added the Hot Keys this has changed... On the other hand it can also be used to add features the game developers forgot, like key re-assignment...

AutoHotKey

This program was written entirely with AutohotKey, link at the bottom of the page.

AutoHotKey is a free to use coding environment specifically designed to control other applications.

HotKeySpeak runs AutoHotKey code through JavaScript, with a small variation on the language, the techniques found on the AutoHotKey forum should still apply with slight modifications. You can visit the AutoHotKey Forum, link at the bottom of the page.

HotKeySpeak's intent with regards to AutoHotKey is to simplify writing AHK code for games and not to circumvent or denigrate this great tool. At the time most of the code was written, AHK V2 was still in Beta.

Known Issues

You do your best but there's always something....

Occasional Interference with copy and paste with Notepad++ where Notepad++ Pastes [NYL] instead of clipboard content - Fix, Restart HotKeySpeak.

Residual speech rule remain active when Application Context changes - Rare, MS SAPI will use the old rule for the first sentence in the new context. The result will be a failure to recognize the first spoken command in this new application context. - Fix, repeat the command. - Fix, this seems to be Microphone Dependent for some reason, so switching microphone may help.

HotKeySpeak stops recognizing voice commands entirely, - Tip, Before starting the game, CTRL + CLICK the application rule in the tree-view and do a test command to make sure the system works beforehand... - Fix, Restart the program. Unplug-reconnect the Mic and restart the program. Run MS SAPI from the options dialog and perform microphone setup. Make sure you did not disable the speech recognition system, see the options window.

Poor recognition, - Tip, run MS SAPI from the options dialog and perform microphone setup and training and ensure the level for the Mic is proper. If you have 2 microphones, make sure that MS SAPI is not using the wrong one.

Actions trigger and I said nothing. - Reason, you may have a noisy setup or you may be playing the game over the speakers and the microphone is picking it up, ambient noise may cause false recognitions for rules using simple one syllable words, you may need to tweak the rule code to bypass the global confidence level, see the HKS.Recognition Example code for overriding confidence level. You may need to use the Recognize When Key is down if your setup is noisy, see the options window.
